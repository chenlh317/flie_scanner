# 家庭数据备份方案（当前设备版）

## 一、当前设备情况

### 数据规模

约 1.5TB：

- 电影
- 照片
- 动画片
- 文档
- Youtube 存档
- 软件安装包

---

### 当前设备

| 设备 | 用途 |
|---|---|
| 2TB 外接 SSD | 主数据盘 |
| 2TB HDD #1 | 本地备份 |
| 2TB HDD #2 | 异地备份 |
| 4TB 联想 NAS（中国） | 非核心补充备份 |
| 电脑内置盘（剩余约300GB） | 临时文件 / 热数据 |

---

## 二、总体思路

采用轻量化的“3-2-1”思路：

- 至少 3 份数据
- 至少 2 种介质（SSD + HDD）
- 至少 1 份异地

但避免：

- NAS 常开
- RAID
- 实时同步
- 复杂自动化系统

目标：

> 简单、稳定、长期可维护。

---

# 三、数据分层

## 1. Critical（最高优先级）

不可替代数据：

```text
/Critical
    Photos
    FamilyVideos
    Documents
    Scan
```

特点：

- 情感价值极高
- 无法重新获取
- 容量相对较小

备份策略：

```text
SSD
+ HDD #1
+ HDD #2
+ NAS（可选）
```

---

## 2. Media（中等优先级）

```text
/Media
    Movies
    Anime
    YoutubeArchive
```

特点：

- 可重新下载
- 但整理成本较高

备份策略：

```text
SSD
+ HDD #1
```

不做云备份。

---

## 3. LowPriority（低优先级）

```text
/LowPriority
    Software
    Temp
```

特点：

- 基本可重建
- 丢失代价低

备份策略：

可仅存在于 SSD。

---

# 四、设备角色定义

| 设备 | 角色 |
|---|---|
| SSD | 唯一主工作盘 |
| HDD #1 | 本地冷备份 |
| HDD #2 | 异地轮换备份 |
| NAS | 关键文件额外副本 |

原则：

> 只有 SSD 是“活数据”。

其余设备：

- 默认断电
- 默认离线
- 仅备份时连接

---

# 五、备份频率

## 推荐频率

### 常规情况

每月一次。

### 特殊情况

在以下情况前额外备份一次：

- 长期离家
- 出国旅行
- 系统重装
- 大量照片整理后

不需要：

- 每周备份
- 实时同步
- NAS 实时镜像

因为数据变化频率不高。

---

# 六、具体备份流程

## 日常状态

```text
SSD 在线
HDD 离线
```

---

## 每月备份流程

### Step 1

连接 HDD #1

运行：

```text
SSD → HDD #1
```

完成后：

- 安全弹出
- 断电收纳

---

### Step 2

连接 HDD #2（异地盘）

运行：

```text
SSD → HDD #2
```

完成后：

- 放回异地
- 不长期保存在电脑旁边

---

### Step 3（可选）

同步关键目录到 NAS：

```text
/Critical
```

NAS 不承担核心备份角色。

---

# 七、FreeFileSync 使用方案

使用软件：

- FreeFileSync

推荐模式：

```text
Update
```

不要使用：

```text
Mirror
```

因为 Mirror 会同步删除。

---

## 推荐设置

### Delete 策略

不要直接删除。

推荐：

```text
Move deleted files to versioning folder
```

---

### Versioning

开启版本历史：

```text
HDD:\_versions\
```

作用：

- 防误删
- 防误覆盖
- 可恢复旧版本

---

## Batch 文件

分别保存：

```text
backup_HDD1.ffs_batch
backup_HDD2.ffs_batch
```

以后只需双击运行。

---

# 八、固定盘符（重要）

FreeFileSync 的 batch 文件依赖路径。

因此应固定盘符。

推荐：

| 设备 | 盘符 |
|---|---|
| SSD | E: |
| HDD #1 | H: |
| HDD #2 | J: |

Windows 设置方式：

```text
此电脑
→ 管理
→ 磁盘管理
→ 更改驱动器号和路径
```

固定后：

- batch 文件长期有效
- 不需要重新配置

---

# 九、关于 SSD 是否改为内置

当前结论：

> 暂不更换为内置 SSD。

原因：

- 当前核心问题不是“内置/外置”
- 而是“是否有稳定多副本体系”

此外：

- 2280 更通用
- 2242 更贵
- 当前外接 SSD 使用没有明显痛点

因此：

> 优先优化备份流程，而不是更换硬件。

---

# 十、关于备份速度

当前速度约：

```text
40 MB/s
```

属于：

> 偏慢但正常的机械 HDD 备份速度。

可能原因：

- 老款 5400rpm HDD
- 大量小文件
- HDD 已写入内圈
- USB 桥接性能一般
- 硬盘老化

对于每月一次备份场景：

并不构成严重问题。

---

# 十一、USB 接口建议

推荐：

```text
SSD 直连电脑
HDD 直连电脑
```

避免：

- 廉价 USB HUB
- 不稳定延长线

不同 USB 口可能略有速度提升。

但对于：

```text
SSD → 单机械 HDD
```

场景：

通常 HDD 本身才是瓶颈。

---

# 十二、长期维护建议

## 每半年

建议：

- 随机打开旧照片
- 测试恢复几个文件
- 检查 HDD SMART 状态

重点不是“是否有备份”。

而是：

> 备份是否真的可恢复。

---

# 十三、最终推荐结构

```text
SSD（主盘）
    ↓
HDD #1（本地冷备份）
    ↓
HDD #2（异地备份）
    ↓
NAS（仅关键文件额外副本）
```

该结构已经足够稳健。

没有必要进一步复杂化。
